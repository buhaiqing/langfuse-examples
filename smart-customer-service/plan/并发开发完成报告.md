## ✅ RAG-03: 混合检索策略实现

### 实现内容

#### 1. BM25Retriever (关键词检索器)
**文件**: `backend/modules/rag_knowledge/retrieval_strategy.py`

**核心功能**:
- ✅ Okapi BM25 排名算法实现
- ✅ 自定义分词器（停用词过滤）
- ✅ IDF (逆文档频率) 计算与缓存
- ✅ 文档索引构建（词频统计）
- ✅ 支持元数据过滤

**技术亮点**:
```python
# BM25 公式实现
Score(q,d) = Σ IDF(qi) * (tf(qi,d) * (k1 + 1)) / (tf(qi,d) + k1 * (1 - b + b * |d|/avgdl))

# 参数可调
- k1: 1.5 (词频饱和参数)
- b: 0.75 (长度归一化参数)
- epsilon: 0.25 (IDF平滑参数)
```

#### 2. MMRReranker (最大边界相关性去重器)
**核心功能**:
- ✅ MMR 算法实现（平衡相关性与多样性）
- ✅ Jaccard 相似度计算
- ✅ Lambda 参数控制（相关性 vs 多样性权衡）
- ✅ 贪婪选择算法

**技术亮点**:
```python
# MMR 公式
MMR = argmax [ λ * Rel(d,q) - (1-λ) * max Sim(d, d_selected) ]

# 默认 lambda = 0.7 (偏向相关性)
```

#### 3. HybridRetriever (混合检索器)
**核心功能**:
- ✅ 三种检索模式: vector, keyword, hybrid
- ✅ Reciprocal Rank Fusion (RRF) 分数融合
- ✅ 可配置权重 (alpha=0.7, beta=0.3)
- ✅ 自动 MMR 重排序
- ✅ 完整的 Langfuse 追踪

**技术亮点**:
```python
# RRF 融合公式
RRF_score(d) = Σ 1 / (k + rank_i(d))
其中 k=60 (标准常数)

# 加权组合
final_score = alpha * vector_rrf + beta * bm25_rrf
```

## 📚 技术文档

### 关键设计决策

1. **为什么选择 BM25 + 向量混合？**
   - BM25 擅长精确关键词匹配
   - 向量擅长语义理解
   - 两者互补，提升召回率和准确率

2. **为什么使用 RRF 而不是简单加权？**
   - RRF 对排名位置敏感，不受绝对分数影响
   - 更鲁棒，不需要分数归一化
   - 业界标准做法 (Elasticsearch, Vespa 等采用)

3. **为什么 MMR lambda 默认 0.7？**
   - 平衡相关性 (70%) 和多样性 (30%)
   - 客服场景更需要准确性
   - 可根据业务调整

4. **为什么熔断器阈值设为 5？**
   - 避免过早触发（偶发失败）
   - 快速检测系统性故障
   - 30秒恢复时间合理
