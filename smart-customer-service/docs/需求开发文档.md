## 1. 项目概述

本项目旨在构建一个基于 Langfuse 的智能客服系统，实现会话路径全链路追踪与高频失败问题分析。通过标准化的埋点体系和多维度的数据分析，提升客服系统的可观测性、问题解决率和用户满意度。

### 1.1 核心价值
- **全链路可观测**: 打破智能客服会话黑盒，从进线到结束完整追踪
- **失败根因定位**: 标准化埋点体系，快速定位问题根因
- **数据驱动优化**: A/B测试验证，量化优化效果
- **降低人工成本**: 提升问题解决率，降低转接率

##### 12. A/B测试框架 (`analysis/ab_testing.py`)
**状态**: ✅ 已完成  
**核心类**:
- `ABTestExperiment` - 实验管理类
- `ExperimentResult` - 实验结果数据类
- `ExperimentGroup` - 实验组枚举 (CONTROL, TREATMENT_A/B/C)

**功能点**:
- ✅ 确定性用户分组 (MD5哈希，保证同一用户始终在同一组)
- ✅ 灵活分配比例 (支持不等分，如50:30:20)
- ✅ 统计显著性检验 (Welch's t-test，不假设等方差)
- ✅ 效应量计算 (Cohen's d)
- ✅ 95%置信区间
- ✅ 功效分析 (计算所需样本量)
- ✅ 多实验对比排序
- ✅ Langfuse元数据集成

**统计方法**:
```python
# Welch's t-test (独立样本t检验)
t_stat, p_value = stats.ttest_ind(
    treatment_array, control_array,
    equal_var=False,  # 不假设等方差
    alternative="two-sided"
)

# Cohen's d (效应量)
pooled_std = sqrt((var_control + var_treatment) / 2)
effect_size = (mean_treatment - mean_control) / pooled_std

# 95%置信区间
ci_lower = mean_diff - 1.96 * se_diff
ci_upper = mean_diff + 1.96 * se_diff
```

**使用示例**:
```python
experiment = ABTestExperiment(
    name="intent_recognition_v2",
    description="Testing improved intent recognition model",
    metric_name="user_satisfaction",
    min_sample_size=50
)

# 分配用户
group = experiment.assign_experiment_group("user_123")

# 记录指标
experiment.record_metric("user_123", 4.5)

# 计算显著性
result = experiment.calculate_statistical_significance()
print(result.summary())

# 样本量计算
n = calculate_required_sample_size(
    baseline_mean=4.0,
    baseline_std=1.0,

    power=0.8
)
```

## 3. Langfuse 集成要点详解

### 3.1 Langfuse 核心概念映射

```
智能客服系统层级          Langfuse 对应概念        说明
─────────────────────────────────────────────────────
一次完整会话              Trace                   绑定 session_id + user_id
意图识别/RAG/工具调用     Span                    业务节点，可嵌套
LLM 调用                 Generation              自动记录 Prompt/Completion/Tokens
关键事件 (升级/失败)      Event                   离散事件标记
质量评估 (满意度等)       Score                   NUMERIC/CATEGORICAL/BOOLEAN
```

### 3.2 追踪层级设计

#### Trace 层级 (会话级)
```python
@trace_customer_service(
    name="customer_service_interaction",
    session_id="session_123",
    user_id="user_456",  # 自动哈希为 a1b2c3d4e5f6g7h8
    tags=["tech_support", "web_chat"],
    metadata={
        "channel": "web_chat",
        "customer_tier": "enterprise",
        "product_version": "v2.3"
    }
)
async def handle_message(user_message, session_id, user_id):
    # 整个函数执行期间的所有操作都属于这个 Trace
    pass
```

**关键点**:
- ✅ `as_type="agent"` - 标识为 Agent 类型 Trace
- ✅ `user_id` 自动 SHA-256 哈希 (前16位)
- ✅ `session_id` 用于关联多轮对话
- ✅ `tags` 用于过滤和分组
- ✅ `metadata` 存储结构化元数据

---

#### Span 层级 (业务节点级)
```python
# 示例: 意图识别 Span
with create_span(
    name="step_1_intent_recognition",
    input_data={"user_message": user_message}
) as span:
    intent_result = await recognize_user_intent(user_message, session_id)
    span.end(
        output_data={
            "intent": intent_result.intent,
            "confidence": intent_result.confidence
        }
    )
```

**标准 Span 命名规范**:
```
step_1_intent_recognition       # 意图识别
step_2_state_update             # 状态更新
step_3_escalation_evaluation    # 升级评估
step_4_normal_processing        # 正常处理
step_4_fallback_response        # Fallback响应
error_handling                  # 错误处理
```

**Span 嵌套示例** (RAG模块):
```
rag_knowledge_query (Span)
├── query_rewriting (Span)
│   └── Generation (LLM call)
├── vector_retrieval (Span)
```

### 3.3 评分体系集成

#### 评分类型
```python
# 1. NUMERIC - 数值型 (0-1 或任意范围)
score_trace("intent_confidence", 0.92, data_type="NUMERIC")
score_trace("response_latency_ms", 1200, data_type="NUMERIC")
score_trace("user_satisfaction", 4.5, data_type="NUMERIC")  # 1-5分

# 2. BOOLEAN - 布尔型 (0 或 1)
score_trace("issue_resolved", 1.0, data_type="BOOLEAN")
score_trace("escalation_required", 0.0, data_type="BOOLEAN")

# 3. CATEGORICAL - 分类别
score_trace("failure_type", "intent_recognition_error", data_type="CATEGORICAL")
score_trace("escalation_reason", "low_confidence", data_type="CATEGORICAL")
```

#### 评分时机
```python
# ✅ 推荐: 在业务逻辑完成后立即评分
async def handle_message(...):
    # 业务逻辑
    result = await process_request()
    
    # 立即评分
    score_issue_resolved(result.is_resolved)
    score_response_latency(result.processing_time_ms)
    
    return result

# ❌ 避免: 延迟评分可能导致 Trace 已关闭
```

```python
# 一次性添加多个评分
```

### 3.6 错误处理与降级

#### 追踪失败不影响业务
```python
@trace_customer_service(...)
async def handle_message(...):
    try:
        # 业务逻辑
        result = await process()
        return result
    except Exception as e:
        # Langfuse 会自动捕获异常并记录到 Trace
        # 即使追踪失败，异常仍会正常抛出
        raise
```

#### 优雅降级
```python
try:
    langfuse.update_current_trace(metadata={...})
except Exception as e:
    # 追踪失败不应影响业务
    logger.warning(f"Failed to update trace: {e}")
    # 继续执行业务逻辑
```

3. **批量处理**: 评分上报改为批量提交
