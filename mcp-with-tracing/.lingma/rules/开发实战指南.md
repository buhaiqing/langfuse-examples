## 1. 常见陷阱与解决方案

### 1.1 ContextVar 可变默认值陷阱

**问题描述** (Task 2):

ContextVar 使用可变默认值(如 None)会导致所有异步任务共享同一个对象,造成状态污染。

**错误示例**:

```python
# 错误:三个问题
# 1. 默认值 None 是单例,所有上下文共享
# 2. 类型注解 list | None,但使用时期望 list
# 3. 没有使用 Token 机制清理

from contextvars import ContextVar

_session_stack: ContextVar[list | None] = ContextVar(
    "session_stack", default=None
)

def push_session(session: dict):
    stack = _session_stack.get()
    if stack is None:
        stack = []
    stack.append(session)
    _session_stack.set(stack)
```

**正确示例**:

```python
# 正确:使用空列表作为默认值,并使用 Token 机制

from contextvars import ContextVar

# 1. 默认值使用不可变类型(空列表)
_session_stack: ContextVar[list] = ContextVar(
    "session_stack", default=[]
)

def push_session(session: dict):
    # 2. 复制列表,避免修改共享对象
    stack = _session_stack.get().copy()
    stack.append(session)
    _session_stack.set(stack)

# 3. 使用上下文管理器 + Token 机制
class SessionContextManager:
    def __init__(self, session: dict):
        self.session = session
        self._token = None

    def __enter__(self):
        stack = _session_stack.get().copy()
        stack.append(self.session)
        self._token = _session_stack.set(stack)
        return self

    def __exit__(self, *args):
        # 4. 使用 Token 恢复到之前的状态
        _session_stack.reset(self._token)
```

**检查清单**:
- [ ] ContextVar 的默认值是否不可变?
- [ ] 类型注解与默认值类型是否一致?
- [ ] 是否复制可变对象避免共享?
- [ ] 是否使用 Token 机制清理?

### 1.2 Mock 路径错误陷阱

**问题描述** (Task 7):

Mock 路径指向错误模块,导致 Mock 不生效,测试仍然调用真实 API。

**错误示例**:

```python
# health.py 文件
from src.observability.instrumentation import get_langfuse_client

def check_health():
    client = get_langfuse_client()

# 错误:Mock 路径错误
with patch("src.observability.health.get_langfuse_client"):
    result = check_health()  # 仍然调用真实的 get_langfuse_client
```

**正确示例**:

```python
# 正确:Mock 实际使用的模块
with patch("src.observability.instrumentation.get_langfuse_client"):
    result = check_health()  # Mock 生效
```

**调试技巧**:

```python
# 如何找到正确的 Mock 路径?

# 1. 查看模块的导入语句
import inspect
import src.observability.health as module

source = inspect.getsource(module)
print(source)
# 找到:from src.observability.instrumentation import get_langfuse_client

# 2. Mock 路径 = 定义该函数的模块路径
```

### 1.3 异步函数中使用阻塞调用

**问题描述** (Task 6):

在异步应用中使用 time.sleep() 或 threading.Thread 会阻塞事件循环。

**错误示例**:

```python
# 错误:阻塞调用冻结事件循环
import threading
import time

class AlertMonitor:
    def start(self):
        self._thread = threading.Thread(target=self._loop, daemon=True)
        self._thread.start()

    def _loop(self):
        while self._running:
            self.check_alerts()
            time.sleep(60)  # 阻塞调用
```

**正确示例**:

```python
# 正确:使用 AsyncIOScheduler 和异步 sleep
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.interval import IntervalTrigger

class AlertMonitor:
    def start(self):
        self._scheduler = AsyncIOScheduler()
        self._scheduler.add_job(
            func=self.check_alerts,
            trigger=IntervalTrigger(minutes=1),
            max_instances=1,
        )
        self._scheduler.start()

    async def check_alerts(self):
        await self._fetch_metrics()
        await self._evaluate_rules()
```

## 2. 测试编写最佳实践

### 2.1 测试离线运行

**目标**: 测试无需 .env 文件或网络连接即可运行。

**错误示例**:

```python
# 错误:测试依赖真实 API
def test_collect_metrics():
    collector = MetricsCollector()
    rate = collector.collect_success_rate()  # 调用真实 API
    assert rate > 0
```

**正确示例**:

```python
# 正确:使用 conftest.py 中的自动 Mock
def test_collect_metrics(mock_langfuse_client, sample_trace_data):
    mock_response = MagicMock()
    mock_response.data = sample_trace_data
    mock_langfuse_client.get_traces.return_value = mock_response

    collector = MetricsCollector()
    rate = collector.collect_success_rate()

    assert rate == 0.8
    mock_langfuse_client.get_traces.assert_called_once()
```

### 2.2 测试数据 Fixtures

**原则**: 提供标准化、可预测的测试数据。

```python
# tests/conftest.py

@pytest.fixture
def sample_trace_data():
    """提供标准测试数据:10 条 traces,80% 成功率"""
    traces = []
    for i in range(10):
        trace = MagicMock()
        trace.id = f"trace_{i}"
        trace.status = "OK" if i < 8 else "ERROR"
        trace.duration = 100 + i * 50
        trace.session_id = "test_session"
        trace.user_id = "test_user"
        traces.append(trace)
    return traces
```

### 2.3 需要真实 API 的测试

**原则**: 使用 @pytest.mark.requires_api 标记,CI/CD 中跳过。


## 3. 性能优化实战案例

### 3.1 缓存 API 调用

**优化前**:

```python
# 错误:每次都调用 API
class MetricsCollector:
    def _fetch_traces(self, session_id: str | None = None) -> list:
        return self._langfuse.get_traces(session_id=session_id, limit=1000)

    def collect_success_rate(self) -> float:
        traces = self._fetch_traces()  # 网络请求
        return success_rate

    def collect_latency_p95(self) -> float:
        traces = self._fetch_traces()  # 又调用一次!重复请求
        return latency
```

**优化后**:

```python
# 正确:使用 TTLCache
from src.observability.cache import TTLCache

class MetricsCollector:
    def __init__(self, window_minutes: int = 10, cache_ttl: int = 300):
        self.window_minutes = window_minutes
        self._trace_cache = TTLCache(ttl=cache_ttl, max_size=64)

    def _fetch_traces(self, session_id: str | None = None) -> list:
        cache_key = f"traces:{session_id or 'all'}:{self.window_minutes}"
        
        cached_traces = self._trace_cache.get(cache_key)
        if cached_traces is not None:
            return cached_traces
        
        traces = self._langfuse.get_traces(session_id=session_id, limit=1000)
        self._trace_cache.set(cache_key, traces)
        return traces
```

**性能对比**:

| 操作 | 优化前 | 优化后 | 改进 |
|------|--------|--------|------|
| 首次调用 | 200ms | 200ms | 无变化 |
| 第二次调用 | 200ms | 1ms | 99.5% 下降 |
| API 调用次数 | 每次 | 5 分钟 1 次 | 减少 99% |

## 4. 异步编程实战模式

### 4.1 后台任务启动与停止

```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.interval import IntervalTrigger

class BackgroundTaskManager:
    def __init__(self, interval_minutes: int = 10):
        self.interval = interval_minutes
        self._scheduler: AsyncIOScheduler | None = None
        self._is_running = False

    def start(self):
        if self._is_running:
            return

        self._scheduler = AsyncIOScheduler()
        self._scheduler.add_job(
            func=self._execute_task,
            trigger=IntervalTrigger(minutes=self.interval),
            max_instances=1,
        )
        self._scheduler.start()
        self._is_running = True

    def stop(self):
        if self._scheduler and self._is_running:
            self._scheduler.shutdown(wait=True)
            self._is_running = False

    async def _execute_task(self):
        try:
            await self._do_work()
        except Exception as e:
            logger.exception("Task failed: %s", e)

    def get_status(self) -> dict:
        return {
            "is_running": self._is_running,
            "interval_minutes": self.interval,
        }
```

### 4.2 测试异步调度器

```python
import pytest

class TestBackgroundTaskManager:
    @pytest.mark.asyncio
    async def test_start_stop(self):
        manager = BackgroundTaskManager(interval_minutes=1)
        manager.start()
        assert manager._is_running is True

        manager.stop()
        assert manager._is_running is False
```

## 5. 代码重构指南

### 5.1 从 threading 迁移到 AsyncIOScheduler

**Step 1: 识别 threading 代码**

```python
# 重构前
import threading
import time

class SmartAlertManager:
    def __init__(self):
        self._monitoring_thread: threading.Thread | None = None
        self._stop_monitoring = False

    def start_monitoring(self):
        self._monitoring_thread = threading.Thread(
            target=self._loop, daemon=True
        )
        self._monitoring_thread.start()
```

**Step 2: 替换为 AsyncIOScheduler**

```python
# 重构后
from apscheduler.schedulers.asyncio import AsyncIOScheduler

class SmartAlertManager:
    def __init__(self, detection_interval_minutes: int = 10):
        self._scheduler: AsyncIOScheduler | None = None
        self._is_running = False

    def start_monitoring(self):
        self._scheduler = AsyncIOScheduler()
        self._scheduler.add_job(
            func=self._run_detection_cycle,
            trigger=IntervalTrigger(minutes=self.detection_interval),
            max_instances=1,
        )
        self._scheduler.start()
        self._is_running = True

    def stop_monitoring(self):
        if self._scheduler and self._is_running:
            self._scheduler.shutdown(wait=True)
            self._is_running = False
```

**Step 3: 更新测试**

```python
# 重构前
def test_monitoring_thread_start_stop():
    manager = SmartAlertManager()
    manager.start_monitoring()
    assert manager._monitoring_thread.is_alive()

# 重构后
@pytest.mark.asyncio
async def test_monitoring_thread_start_stop():
    manager = SmartAlertManager()
    manager.start_monitoring()
    assert manager._is_running is True
```

**关键变化**:
1. threading.Thread -> AsyncIOScheduler
2. time.sleep() -> IntervalTrigger
3. _monitoring_thread -> _scheduler
4. 测试添加 @pytest.mark.asyncio
