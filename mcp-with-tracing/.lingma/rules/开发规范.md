## 1. 数据安全与隐私保护

### 1.1 ContextVar 使用规范

**背景**: Task 2 修复了 ContextVar 可变默认值导致的状态污染问题。

#### 强制规则

**规则 1.1.1**: ContextVar 默认值必须使用不可变类型

```python
# 错误:可变默认值会被所有上下文共享
_session_stack: ContextVar[list] = ContextVar("session_stack", default=None)

# 正确:使用空列表作为默认值
_session_stack: ContextVar[list] = ContextVar("session_stack", default=[])
```

**规则 1.1.2**: 类型注解必须与默认值类型一致

```python
# 正确:类型注解与默认值匹配
_ctx: ContextVar[list] = ContextVar("ctx", default=[])
_ctx: ContextVar[str | None] = ContextVar("ctx", default=None)
```

**规则 1.1.3**: 必须在上下文管理器中使用 Token 机制清理

```python
# 正确:使用 Token 确保正确清理
class TraceContextManager:
    def __enter__(self):
        self._token = _trace_id_context.set(self.trace_id)
        return self

    def __exit__(self, *args):
        _trace_id_context.reset(self._token)
```

**检查清单**:
- [ ] 所有 ContextVar 的默认值是否不可变?
- [ ] 类型注解与默认值类型是否一致?
- [ ] 是否使用了 Token 机制进行清理?
- [ ] 是否在 end_trace() 时重置为初始状态?

```python
# 正确:在 Langfuse 客户端配置全局脱敏
langfuse = Langfuse(
```

**规则 1.2.2**: 必须脱敏的字段列表

| 字段类型 | 示例字段 | 脱敏规则 |

```python

import hashlib

def hash_user_id(user_id: str) -> str:
    return hashlib.sha256(user_id.encode()).hexdigest()[:16]
```

## 2. 异步编程与并发控制

### 2.1 任务调度器选择

**背景**: Task 6 统一了异步监控架构,替换 threading 为 AsyncIOScheduler。

#### 强制规则

**规则 2.1.1**: 异步应用必须使用 AsyncIOScheduler,禁止使用 threading

```python
# 正确:使用 AsyncIOScheduler(与 FastAPI 事件循环兼容)
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.interval import IntervalTrigger

class AlertMonitor:
    def start_monitoring(self):
        self._scheduler = AsyncIOScheduler()
        self._scheduler.add_job(
            func=self._run_detection_cycle,
            trigger=IntervalTrigger(minutes=10),
            max_instances=1,
        )
        self._scheduler.start()

# 错误:使用 threading.Thread(会阻塞事件循环)
import threading

class AlertMonitor:
    def start_monitoring(self):
        self._thread = threading.Thread(target=self._monitoring_loop, daemon=True)
        self._thread.start()
```

**规则 2.1.2**: 调度任务必须设置 max_instances=1

```python
# 正确:防止任务重叠执行
scheduler.add_job(
    func=heavy_task,
    trigger=IntervalTrigger(minutes=5),
    max_instances=1,
)
```

**规则 2.1.3**: 必须在应用关闭时正确关闭调度器

```python
# 正确:优雅关闭
def stop_monitoring(self):
    if self._scheduler and self._is_running:
        self._scheduler.shutdown(wait=True)
        self._is_running = False
```

**检查清单**:
- [ ] 是否使用了 AsyncIOScheduler 而非 threading?
- [ ] 所有定时任务是否设置了 max_instances=1?
- [ ] 是否在应用关闭时调用了 scheduler.shutdown(wait=True)?
- [ ] 是否有测试验证调度器生命周期?

### 2.2 并发控制

**规则 2.2.1**: 批量操作必须使用信号量限制并发

```python
# 正确:使用信号量控制并发数
async def process_batch(items: list, max_concurrent: int = 10):
    semaphore = asyncio.Semaphore(max_concurrent)
    
    async def limited_process(item):
        async with semaphore:
            return await process_item(item)
    
    tasks = [limited_process(item) for item in items]
    return await asyncio.gather(*tasks)
```

**检查清单**:
- [ ] 批量操作是否使用了信号量?
- [ ] 并发数是否合理设置(通常 5-20)?
- [ ] 是否处理了任务异常(return_exceptions=True)?

**背景**: Task 7 创建了全局 conftest.py fixtures,实现了测试离线运行。

#### 强制规则

**规则 3.1.1**: 所有外部依赖必须通过 conftest.py 自动 Mock

```python
@pytest.fixture(autouse=True)
def mock_langfuse_client():
    mock_client = MagicMock()

    with patch("src.observability.instrumentation._langfuse_client", mock_client):
        yield mock_client
```

**规则 3.1.2**: 需要真实 API 的测试必须使用 @pytest.mark.requires_api

```python
@pytest.mark.requires_api
def test_real_langfuse_connection():
    client = get_langfuse_client()
    assert client is not None

# CI/CD 中跳过 requires_api 测试

```

**规则 3.1.3**: Mock 路径必须指向实际导入的模块

```python
# 正确:Mock 实际使用的模块
with patch("src.observability.instrumentation.get_langfuse_client"):

    pass
```

## 4. 性能优化与缓存策略

### 4.1 API 调用缓存

**背景**: Task 5 实现了 TTLCache,减少 80%+ API 调用。

#### 强制规则

**规则 4.1.1**: 重复的 API 调用必须使用 TTL 缓存

```python
# 正确:使用 TTLCache 缓存 API 结果
class MetricsCollector:
    def __init__(self, cache_ttl: int = 300):
        self._trace_cache = TTLCache(ttl=cache_ttl, max_size=64)

    def _fetch_traces(self, session_id: str | None = None) -> list:
        cache_key = f"traces:{session_id or 'all'}:{self.window_minutes}"
        
        cached = self._trace_cache.get(cache_key)
        if cached is not None:
            return cached
        
        traces = self._langfuse.get_traces(session_id=session_id)
        self._trace_cache.set(cache_key, traces)
        return traces
```

**规则 4.1.2**: 缓存键必须包含所有影响结果的参数

```python
# 正确:缓存键包含所有参数
cache_key = f"traces:{session_id or 'all'}:{self.window_minutes}"

# 错误:缓存键不完整
cache_key = f"traces:{session_id}"
```

**规则 4.1.3**: 必须提供缓存统计和失效机制

```python
def get_cache_stats(self) -> dict:
    return {
        "hit_rate": self._trace_cache.hit_rate,
        "size": self._trace_cache.size,
    }

def invalidate_cache(self, key: str | None = None):
    if key:
        self._trace_cache.delete(key)
    else:
        self._trace_cache.clear()
```

**检查清单**:
- [ ] 重复的 API 调用是否使用了缓存?
- [ ] 缓存键是否包含所有影响结果的参数?
- [ ] 是否提供了缓存统计方法?
- [ ] 是否提供了手动失效机制?
- [ ] 缓存 TTL 是否可配置?
