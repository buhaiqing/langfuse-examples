# Makefile 一键启动命令使用指南

## 🚀 快速启动

### 方式 1: 一键启动所有服务（推荐）⭐

```bash
make start-all
```

**这个命令会：**
1. 🔍 **检查并停止已运行的服务（幂等性保证）**
2. ✅ 检查系统环境（操作系统、架构、时间）
3. ✅ 检查 Python 环境（版本、路径、uv）
4. ✅ 检查依赖包（Reflex、Langfuse、FastMCP）
5. ✅ 检查环境变量（Langfuse API Keys、OpenAI Key）
6. ✅ 检查端口状态（3000、8001、8000）
7. 🚀 **分步启动服务（带超时控制）**
8. ✅ **验证服务是否真正启动**
9. 📄 **输出日志文件路径**

### 💡 容错特性

1. **优雅降级**：SIGTERM → 等待 3s → SIGKILL（强制终止）
2. **超时控制**：Reflex UI 最多等待 15 秒，MCP Server 最多等待 10 秒
3. **启动验证**：每秒检查端口，确认服务真正启动
4. **自动清理**：启动失败时自动清理残留进程
5. **日志记录**：所有输出保存到 `/tmp/` 目录，便于排查
6. **错误提示**：失败时显示具体原因和解决建议

## 🔧 故障排查

### 问题 1: 端口被占用

如果看到：
```
• Reflex UI (3000): ❌ 已被占用
```

**解决方案：**

```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或者修改端口（编辑 rxconfig.py）
config = rx.Config(
    app_name="app",
    frontend_port=3001,  # 改为其他端口
)
```

### 问题 2: 环境变量未配置

如果看到：
```
• LANGFUSE_PUBLIC_KEY: ❌ 未配置
• LANGFUSE_SECRET_KEY: ❌ 未配置
```

**解决方案：**

```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env 文件，填入实际的 API Keys
vim .env
```

### 问题 3: 依赖包未安装

如果看到：
```
• Reflex: ❌ 未安装
```

**解决方案：**

```bash
# 安装 Reflex
make ui-install

# 或手动安装
uv pip install reflex
```

## 🎯 常用工作流

### 首次使用

```bash
# 1. 安装依赖
make install
make ui-install

# 2. 配置环境变量
cp .env.example .env
vim .env  # 编辑填入 API Keys

# 3. 一键启动
make start-all
```

### 日常开发

```bash
# 启动所有服务
make start-all

# 或者分别启动（两个终端）
# 终端 1:
make start

# 终端 2:
make ui
```

### 调试模式

```bash
# 快速测试（仅运行单元测试）
make dev

# 监视文件变化自动测试
make watch
```
