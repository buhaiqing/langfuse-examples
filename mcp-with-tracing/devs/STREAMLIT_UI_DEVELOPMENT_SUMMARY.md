## 关键设计决策

### 1. 零 API 开发
- **决策**: 直接导入项目模块，不开发额外的 REST API
- **优势**: 快速开发、代码复用、无额外维护成本
- **实现**: 通过 `ui/utils/data_loader.py` 统一数据访问层

### 2. 复用现有缓存机制
- **决策**: 使用项目已有的 TTLCache
- **优势**: 避免重复 API 调用，提升性能
- **实现**: MetricsCollector 内置 5 分钟 TTL 缓存

### 3. 只读展示为主
- **决策**: 大部分数据只读，仅告警规则和提示词版本支持 CRUD
- **原因**: 符合监控仪表板的典型使用场景
- **例外**: 允许创建告警规则和注册提示词版本

### 4. 独立运行
- **决策**: UI 与 MCP Server 独立运行
- **优势**: 互不影响，可独立部署
- **实现**: Streamlit 默认端口 8501，MCP Server 使用其他端口

## 已知限制

1. **单进程**: Streamlit 单进程模型，多用户并发需注意性能
2. **数据实时性**: 依赖缓存 TTL，非完全实时（5 分钟延迟）
3. **ML 依赖**: 智能告警需要 Prophet、PyOD 等可选依赖
4. **本地运行**: 默认仅本地访问，生产环境需额外配置
