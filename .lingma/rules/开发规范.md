## 3. 错误处理标准

### 强制模式

```python
from core.langfuse_client import langfuse
from core.tracing import observe

class ObservabilityError(Exception):
    """可观测性错误的基类异常。"""
    pass

class TraceSubmissionError(ObservabilityError):
    """当向 Langfuse 提交追踪失败时抛出。"""
    pass

class ValidationError(Exception):
    """当输入验证失败时抛出。"""
    pass

@observe(name="process-request", as_type="span")
def process_request(request: dict) -> dict:
    """
    处理请求并进行适当的错误处理和追踪。

    所有异常会自动被捕获到追踪中。
    """
    try:
        # 验证输入
        if not request.get("session_id"):
            raise ValidationError("session_id 是必需的")

        # 处理请求
        result = perform_operation(request)

        return {"success": True, "data": result}

    except ValidationError as e:
        # 验证错误是预期的,记录 INFO 级别日志
        logger.info(f"验证失败: {e}")
        raise

    except TraceSubmissionError as e:
        # 可观测性错误不应导致操作失败
        logger.error(f"提交追踪失败: {e}")
        # 继续执行 - 优雅降级
        return perform_operation_without_tracing(request)

    except Exception as e:
        # 意外错误必须被捕获并重新抛出
        logger.exception(f"处理请求时发生意外错误: {e}")
        raise
```

### 规则

- ✅ **始终**在通用 `Exception` 之前捕获特定异常
- ✅ **始终**记录带有上下文的异常
- ✅ **始终**重新抛出异常,除非有意进行优雅降级
- ❌ **禁止**使用裸 `except:` 子句
- ❌ **禁止**静默抑制异常
- ❌ **禁止**捕获 `BaseException`(包括 KeyboardInterrupt)

### 脱敏规则

```python
def _mask_value_by_type(field_name: str, value: Any) -> Any:
    """根据字段类型脱敏值"""
    if not isinstance(value, str):
        return "***MASKED***"

    field_lower = field_name.lower()

    # 手机号脱敏: 138****5678
    if field_lower in ["phone", "mobile", "telephone"]:
        if len(value) >= 7:
            return value[:3] + "****" + value[-4:]
        return "***"

    # 邮箱脱敏: zh***@example.com
    if field_lower in ["email", "mail"]:
        parts = value.split("@")
        if len(parts) == 2 and len(parts[0]) > 2:
            return parts[0][:2] + "***@" + parts[1]
        return "***@***"

    # 身份证脱敏: 110101********1234
    if field_lower in ["id_card", "id_number", "identity"]:
        if len(value) >= 10:
            return value[:6] + "********" + value[-4:]
        return "***"

    # 姓名脱敏: 张*
    if field_lower in ["name", "username", "user_name"]:
        if len(value) > 1:
            return value[0] + "*" * (len(value) - 1)
        return "*"

    # 密码/密钥/令牌 - 完全隐藏
    if any(pattern in field_lower for pattern in ["password", "secret", "token", "key"]):
        return "***REMOVED***"

    return "***MASKED***"
```

### 用户 ID 哈希

```python
import hashlib

def hash_user_id(user_id: str) -> str:
    """
    为用户 ID 创建单向哈希以保护隐私,
    同时保持一致性以便追踪

    Args:
        user_id: 原始用户 ID

    Returns:
        哈希后的用户 ID (SHA-256,前16位)
    """
    return hashlib.sha256(user_id.encode()).hexdigest()[:16]
```

### 敏感字段列表

以下字段会被自动脱敏:
- `phone`, `mobile`, `telephone` - 电话号码
- `email`, `mail` - 邮箱地址
- `id_card`, `id_number`, `identity` - 身份证号
- `name`, `username`, `user_name` - 姓名
- `card_number`, `credit_card`, `bank_account` - 银行卡号
- `password`, `secret`, `token`, `api_key` - 密钥类
- `address`, `ip_address`, `ip` - 地址类

### 规则

- ✅ **始终**在发送到 Langfuse 前脱敏 PII (个人身份信息)
- ✅ **始终**对用户 ID 使用哈希而非明文
- ✅ **始终**使用环境变量存储 API 密钥
- ❌ **禁止**记录密码或 API 密钥
- ❌ **禁止**在日志中记录完整的请求/响应体

## 6. Langfuse 追踪规范

### Trace 层级设计

```
Trace (一次完整会话)
├── Span (意图识别)
│   └── Generation (LLM 调用)
├── Span (RAG 检索)
│   └── Event (检索结果)
├── Span (工具调用)
│   └── Generation (工具执行)
└── Score (用户满意度)
```

### 装饰器使用

```python
from core.tracing import trace_customer_service, create_span, score_trace

@trace_customer_service(
    name="api_error_troubleshooting",
    session_id="session_123",
    user_id="user_456",
    tags=["technical_support", "api_issue"],
    metadata={"channel": "web_chat", "priority": "high"}
)
async def handle_api_error(query: str):
    """处理 API 错误的客服对话"""
    
    # 创建 Span
    with create_span("intent_recognition", input_data={"query": query}) as span:
        intent = await recognize_intent(query)
        span.end(output_data={"intent": intent, "confidence": 0.92})
    
    # 评分
    score_trace("intent_confidence", 0.92, comment="高置信度识别")
    
    return {"response": "解决方案...", "escalated": False}
```

### 评分体系

| 评分项 | 类型 | 范围 | 说明 |
|-------|------|------|------|
| `intent_confidence` | NUMERIC | 0-1 | 意图识别置信度 |
| `retrieval_relevance` | NUMERIC | 0-1 | RAG 检索相关性 |
| `tool_success_rate` | NUMERIC | 0-1 | 工具调用成功率 |
| `issue_resolved` | BOOLEAN | 0/1 | 问题是否解决 |
| `user_satisfaction` | NUMERIC | 1-5 | 用户满意度 |
| `response_latency_ms` | NUMERIC | ms | 响应延迟 |
| `failure_type` | CATEGORICAL | - | 失败类型分类 |
| `escalation_required` | BOOLEAN | 0/1 | 是否需要转接 |
